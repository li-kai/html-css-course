HTML

HTML & CSS are file types. Just like jpeg or docx, they tell computers what to show, and how to show them.

Why learn HTML & CSS?

HTML & CSS make the foundation of the world wide web. Every time you visit google.com or instagram.com, what you're loading is their html and css files.
By learning these, you can make your very own website.

Writing code

Code is precise. Content is not.
Before we type any code, I'd just like to point out that code, the parts that computers understand, are precise. Extra symbols or typos will cause errors. If you see any green squigly line, that probably means you typed something the computer doesn't understand. However, for content, the parts where I describe myself, is not. Feel free to type in anything you want, and make the portfolio truly yours.

index.html

This is the html file that your web browser reads and displays.

HTML tags

Open and close
<html> </html>
<button> </button>
<img/>
<input/>
HTML is made up of tags. A tag for most cases, must be opened and closed. There are rare cases where they are self closing, most commonly the img and input tags.

head and body

<head></head>
Metadata
<body></body>
Visual content
I like to think of the structure of html files like a person dancing. You start with the head, which contains all the information, the metadata, about how it is going to perform. The body, which everyone else sees, then displays the moves and actions.

head and body


            <!DOCTYPE html>
            <html>
              <head>
                <title>Portfolio</title>
              </head>
              <body>
                Hi I'm Kai
              </body>
            </html>
Further reading:
  • html tags
Here this is the most basic html we can have. We add a title, which appears at the top of the browser. The body, which shows as content. We can edit this and play run, and see them change as expected. Things look a little ugly right now because we're not doing any styling, yet.

Adding headings


            <!DOCTYPE html>
            <html>
              <head>
                <title>Portfolio</title>
              </head>
              <body>
                <h1>Hi I'm Kai</h1>
                <h2>CEO of Nothing</h2>
              </body>
            </html>
Further reading:
  • h1 to h6
Here we can see the roots of html as a document format, because we have terms like headers, headings and footers. Things used in the printing industries like newspapers. A heading is something used to display titles and to show importance. Headings range from h1 to h6.

Adding images


            <!DOCTYPE html>
            <html>
              <head>
                <title>Portfolio</title>
              </head>
              <body>
                <h1>Hi I'm Kai</h1>
                <h2>CEO of Nothing</h2>
                <img src="./img/building.jpg" alt="building"/>
              </body>
            </html>
Further reading:
  • img tag
  • relative paths
In order to show images, we use the img tag. In the tag, we specific a "src" attribute, which points to a file. If your file is in some other place, you might need to change the file name. It is good practice to add an "alt" tag to describe your image.

Adding CSS


            <!DOCTYPE html>
            <html>
              <head>
                <title>Portfolio</title>
                <link rel="stylesheet" href="style.css" />
              </head>
              <body>
              ...
Further reading:
  • link tag
Things are going to get a lot prettier. We're finally adding in some CSS styles! To start, we need a link tag in the head tag. This tells the browser to download the CSS stylesheet when it reads this HTML file. Remember that the head doesn't no any presentation, so you shouldn't see anything new appear in your browser.

Writing CSS


            body {
              background: teal;
            }
Further reading:
  • MDN: CSS guide
  • CSS property: background
Let's write our first CSS property. We first have selectors, targeting which HTML elements we want, then properties describing how they should look. Here, we're saying we want the background of the body to be teal.

More CSS


            <!DOCTYPE html>
            <html>
              <head>
                <title>Portfolio</title>
                <link rel="stylesheet" href="style.css" />
                <link rel="stylesheet" href="https://stackpath.bootstrapcdn.com/bootstrap/4.3.1/css/bootstrap-reboot.min.css" />
              </head>
              <body>
              ...
Further reading:
  • link tag
The next thing we're going to do is apply a base layer of styles. This helps to normalize our CSS across different browsers, fixes some visual bugs and generally provides a good foundation for us to work with. The moment you add this and run it, your HTML should have changed.
